# ТЗ: Plain-Text API для RubyMUD `#webfetch`

## Цель

Нужен простой HTTP API для сайта `rmud.bssx.ru`, чтобы MUD-клиент RubyMUD мог показывать данные о предметах прямо в игровом окне через встроенную команду:

```text
#alias {лор} {#webfetch {https://rmud.bssx.ru/items.txt} {q} {%0}}
лор мифриловый меч
```

RubyMUD будет делать безопасный `GET` и выводить body ответа как plain text local echo.

## Требуемый Endpoint

```text
GET /items.txt?q=<query>
```

Пример:

```text
https://rmud.bssx.ru/items.txt?q=мифриловый+меч
```

Требования:

1. Ответ должен быть `text/plain; charset=utf-8`.
2. Ответ должен быть человекочитаемым, уже готовым для вывода в MUD-клиент.
3. HTML/Markdown/ANSI не нужны для первой версии.
4. Endpoint должен работать без авторизации.
5. `q` должен искать предметы так же, как текущий `/items.json?q=...`.
6. Если `q` пустой, вернуть короткую usage-строку.
7. Если ничего не найдено, вернуть понятное сообщение.
8. Ограничить вывод по умолчанию, например первыми 5 предметами.
9. Если найдено больше, указать общее число и что показаны только первые N.

## Формат Ответа

Рекомендуемый формат:

```text
rmud-items: found 12, showing 5 for "мифриловый меч"

1. мифриловый меч [#395] (weapon)
  type: оружие Материал: мифрил Вес: 16 Цена: 200
  vli: АТАКА на 10, ВРЕД на 1
  eff: НЕТ
  dmg: 3К5, среднее наносимое повреждение: 9.0.
  prop: МАГИЧЕСКИЙ
  lim: НЕ ДЛЯ МАГОВ, НЕ ДЛЯ ЖРЕЦОВ
  req: режущее оружие

2. ...
```

Поля показывать только если они заполнены.

Приоритетные поля:

1. `name`
2. `id`
3. `item_type`
4. `tip`
5. `vli`
6. `eff`
7. `dmg`
8. `armor`
9. `prop`
10. `lim`
11. `req_skill`
12. `mater`
13. `knowledge`, только если короткое или отдельным параметром

## Single Item by ID

Желательно добавить:

```text
GET /items/:id.txt
```

Пример:

```text
https://rmud.bssx.ru/items/395.txt
```

Поведение: вернуть полную plain-text карточку одного предмета.

## Optional Параметры

Для первой версии достаточно только `q`, но было бы полезно поддержать:

```text
GET /items.txt?q=кольцо&limit=10
GET /items.txt?q=меч&vli=АТАКА
GET /items.txt?q=меч&eff=СВЯТОЙ
GET /items.txt?q=меч&req_skill=режущее
GET /items.txt?q=кольцо&item_type=ring
```

Если параметры уже есть в `/items.json`, желательно использовать те же имена.

`limit` желательно ограничить сверху, например `20`, чтобы MUD-клиент не получал слишком много строк.

## Ошибки

Ошибки тоже plain text:

```text
rmud-items: query is empty
```

```text
rmud-items: nothing found for "..."
```

```text
rmud-items: internal error, try later
```

Для пользовательских ошибок можно возвращать `200 OK` с текстом. Для настоящих серверных ошибок лучше `5xx`, но body всё равно text/plain.

## Ограничения для Совместимости с RubyMUD

RubyMUD `#webfetch` ожидает:

1. Только `https://`.
2. Метод только `GET`.
3. Ответ plain text.
4. UTF-8.
5. Разумный размер ответа: лучше держать меньше `64 KiB`.
6. Разумное число строк: лучше держать меньше `200`.
7. Быстрый ответ: желательно до `5s`.

RubyMUD сам URL-encode'ит значение `q`, поэтому API должен принимать обычный query parameter.

## Пример RubyMUD Интеграции

После реализации endpoint пользователь сможет создать alias:

```text
#alias {лор} {#webfetch {https://rmud.bssx.ru/items.txt} {q} {%0}}
```

И использовать:

```text
лор мифриловый меч
лор красное кольцо
лор 395
```

Если хочется поддержать поиск по ID через тот же endpoint, можно сделать:

```text
GET /items.txt?q=395
```

И если `q` является числом, возвращать карточку предмета с таким ID.

## Причина, Почему Не JSON

JSON API уже полезен для программ, но RubyMUD `#webfetch` выводит response body прямо в игровое окно.

Plain-text endpoint нужен как “сайт-плагин”: сайт сам форматирует результат для игрока, а MUD-клиент только безопасно получает и показывает текст.

Это переносимо и работает в Docker без Ruby, curl или локальных скриптов.
