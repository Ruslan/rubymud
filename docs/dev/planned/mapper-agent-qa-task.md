# QA-задача агенту-тестеру — Mapper, ре-тест #4 (`mud_path` доходит внутрь single-exit целей)

**Дата:** 2026-07-16. **Кому:** LLM-агенту, играющему на RMUD через MCP. **От:** оркестратора
сборки. **Companion:** `mapper.md` (§5 трекер, §7 тулы), твои ревью в `mapper-agent-review.md`.
Эта задача **заменяет** задачу ре-теста #3 (тот этап принят с одним хвостом — см. ниже).

## Контекст — что в этой сборке нового

Ре-тест #3 подтвердил: per-edge направления чинены (банк `с`, Гостиница `в`, Q2 `ю;в;ю`), ноль
шагов в стену на инварианте. Но всплыл **один баг**: многохоповый `to_hint` до **single-exit
тупика** (банк, зал, рента) ронял финальный поворот и приходил на **соседа**, не внутрь.

**Root cause и фикс (этой сборки):** guard reverse-consistency из прошлого раунда был слишком
строгим — требовал, чтобы **оба** конца ребра объявляли связь. У тупиков с асимметричными
данными карты (сосед в `.mm2` не записал прямую связь — частое в реальных архивах) финальное
ребро `сосед→цель` резалось. Новый предикат `ConnectsTo`: ребро валидно, если **A явно
объявляет выход, ИЛИ цель B явно объявляет обратный выход, ИЛИ оба permissive** — корректная
двусторонняя связность; вход в тупик восстанавливается по его же back-link, при этом ложные
смещённые рёбра по-прежнему отвергаются. Изменения только в роутере `mud_path`; трекер/парсер
не тронуты.

> ⚠️ Фикс в дереве, применится после пересборки. **Сначала `make run`.**

## Setup
```
make run MUD=<хост:порт>
```
Активный набор на сессии 1 (`rmud`, 81 зона / 8034 комнаты). Для позиции: `look` или шаг →
авто-ресинк 🟢.

## Что проверить

### 1. PRIMARY — `mud_path` доходит ВНУТРЬ single-exit целей (то, что падало)
Проложи маршрут **ИЗДАЛЕКА** и **пройди вживую** через `mud_send_command` до конца:

| Цель (single-exit) | Ожидаемый хвост | Должен зайти В клетку |
|---|---|---|
| `to_hint "Городской банк"` | `…з; с` | (−5,−8) — внутрь банка, не на «Тропинку» |
| `to_hint "Южная часть зала"` | `…в; ю` | (2,−5) — внутрь зала |
| до **ренты** | — | внутрь, без ручного добивания последних хопов |

Критерий PASS: последняя команда реально заводит В целевую комнату (сверь `mud_where` после
последнего шага = целевая клетка), а не оставляет на соседе.

### 2. Backfill — не покрытое ре-тестом #3 (закрыть хвост фазы 2+3)
- **Флаги дверей:** маршрут через реально закрытую дверь (склад `(N)`) → хоп помечен
  `[DOOR — open first]`, в сводке счётчик. Проверь, что не ведёт молча сквозь закрытую.
- **U/D-глаголы:** если ходишь вверх/вниз — `вв/вверх/up`, `вн/вниз/down` ловятся в
  dead-reckoning; `подняться/опуситься` пока НЕ ловятся (best-effort, придёт с `move_words`) —
  это не баг; подтверди лишь, что позиция при этом остаётся верной (ре-анкор по отпечатку).

### 3. Watch — не переоткрылся ли displaced-баг (важно, guard ослаблен)
Фикс **релаксировал** проверку рёбер. Убедись обратным: (а) цели, что раньше были
«недостижимы» из-за строгого guard, **теперь достижимы**; (б) при этом **не появились ложные
рёбра** — на пройденных маршрутах НЕТ шагов «Вы не можете идти» / в стену. Если увидишь новый
шаг-в-стену — зафлажь маршрут (это регресс релаксации).

### 4. Регрессии — не должны деградировать
- **Pipe-collapse:** прогоны схлопнуты (`[pipe run: N cells]`), трекер 🟢 на дальней клетке.
- **Трекер:** `mud_where`/`mud_look_map` 🟢 всю проходку, авто-ресинк, head-cancel на «не
  можете идти»/«Дверь закрыта».

## Архитектурная рамка
**Истина позиции — фингерпринт комнаты, а не детект слова-хода.** «Не распозналось слово» ≠
«потерялся»: трекер ре-анкорится по отпечатку на каждом `[ Exits: ]`-блоке.

## Как отчитаться
Структурно в `mapper-agent-review.md`, секция «Ре-тест #4». По пунктам 1–4: PASS/FAIL + для
FAIL конкретика (маршрут, ожидал/получил, клетки/`ch`). Финал: **закрывается ли фаза 2+3
полностью** (`mud_path` пригоден для «ходить по маршруту» до любой цели) или остаётся хвост.
